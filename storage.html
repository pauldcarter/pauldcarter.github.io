Data & storage

Storage is where I started — my career began as a DBA — and it's still the first decision I get right in any project. The right store ranges from a shared spreadsheet to a geo-enabled Postgres, and picking the smallest one that fits is most of the battle.

Store Best for Pros Cons Where I've used it
Google Sheets the spreadsheet as a database Capture & prototyping Small shared datasets where the users already live in a spreadsheet. Zero setup, humans can edit Built-in sharing, history and an API — the UI comes for free. Not a real database No schema or constraints; slow and fragile past a few thousand rows. Quick tools & early data capture The honest starting point for many data problems — graduate off it when it creaks.
JSON file store a file on disk, done carefully Small curated datasets Config-sized data owned by one process — a database adds nothing. Diffable, portable, zero ops Readable in any editor, versionable in git, trivially backed up. One writer, no queries Concurrency and search are on you; corruption risk without care. Golf Coach Atomic write-and-rename to knowledge-base.json — safe saves without a DB.
Browser storage localStorage / IndexedDB in a PWA Single-user, on-device data Personal apps where the data belongs to the person, not a server. Private and offline by default Works with no network and no account; nothing to host or secure. Trapped on the device No cross-device sync; clearing site data deletes everything. Family Manager An installable PWA whose data never leaves the device.
SQLite a real SQL database in one file Single-server apps Full SQL with transactions, without running a database server. Zero DB ops, easy backups The database is a file; copy it and you've backed it up. One box, limited concurrency Writers serialise; no network access from other machines. Retail Platform Right-sized for a shop's load — web, till and kiosk on one stock pool.
Cloudflare D1 SQLite at the edge Workers apps needing SQL Relational data colocated with edge compute — no separate DB host. Serverless SQL, generous free tier Migrations, backups and bindings built into the Workers platform. Young, size-capped Per-database size limits; fewer tools than a grown-up Postgres. RunTracker Run results and share links, queried from the same Worker that serves the app.
KV + Durable Objects edge key-value & live state Sessions & real-time state KV for read-heavy persistence; a Durable Object per game/room for live state. Built for the edge DO gives single-threaded consistency per object — multiplayer without locks. Not relational No joins or ad-hoc queries; KV is eventually consistent across regions. Learning Lab Durable Objects hold live multiplayer Perudo games; KV persists the rest.
MySQL the classic relational workhorse Shared relational workloads Multi-user OLTP with a mature replication and tooling story. Proven at every scale Runs everywhere, hires everywhere; decades of operational knowledge. A server to own Provisioning, tuning, backups and upgrades — someone has to be the DBA. Career database work Where I learned the discipline: schemas, indexing and query plans as a DBA.
Postgres on Supabase managed Postgres + PostGIS Data products & geo queries The most capable open-source database, with the ops handled for you. PostGIS, RLS, instant REST Geo queries out of the box and an API generated straight off the schema. Platform-shaped access PostgREST row caps mean pagination; free tier pauses when idle. Schoolwise School locations and multi-year results, with a byte-for-byte raw audit layer.
