Games

Casual browser games built to learn how games actually work — game loops, collision, state. They run natively in any browser with nothing to install, which makes them perfect for a Chromebook.

Visit live ↗

Twin JavaScript + Python implementations

Each game is written twice, in the same shape, in both languages. Put the two versions side by side and the game logic reads through the syntax — a deliberate teaching device, not just a portfolio of games.

Key features

  • Every game implemented twice — JavaScript for the browser, Python alongside — structured identically for side-by-side reading.
  • Pure vanilla JavaScript: no frameworks, no build step, no install.
  • Runs on anything with a browser, including locked-down Chromebooks.
  • Fully static — no server, no database; game state lives in the browser during play.
  • Deploys on every push via GitHub Pages.

How it's built
