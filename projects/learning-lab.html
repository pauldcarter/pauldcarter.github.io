Learning Lab

An EdTech platform for kids: mental maths, spelling and story-writing apps, plus a real-time multiplayer Perudo (liar's dice) game — all on one shared shell, running entirely at the edge.

Visit live ↗

Learning apps + turn-based multiplayer, one platform

Most kids' learning sites are single-player worksheets. Learning Lab pairs practice apps with a genuinely real-time multiplayer game — live state held in Durable Objects over WebSockets — so the same account and shell cover both homework and play.

Key features

  • Mental maths, spelling and story-writing apps for kids, on a single SPA shell with shared sign-in.
  • Real-time multiplayer Perudo — live games over WebSockets, with each game's state in its own Durable Object.
  • Google sign-in with CSRF-protected OAuth and session cookies.
  • Hand-rolled routing in a Python Cloudflare Worker — no framework — for a lean bundle and fast cold starts.
  • No traditional database: Durable Objects hold live state, KV handles persistence.

How it's built

Front end

Vanilla JS SPA shell + router Lean client, no framework weight.

Back end

Python Cloudflare Worker Hand-rolled routing for a lean, fast-cold-start edge bundle.

APIs / Auth

REST + WebSockets · Google OAuth Real-time play over Durable Objects; CSRF-protected sign-in with session cookies.

Hosting

Cloudflare Workers Edge runtime, instant deploys.

Database

KV + Durable Objects DO holds live multiplayer state; KV for persistence — no Postgres needed.
