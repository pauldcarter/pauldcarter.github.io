RunTracker

Parkrun gives you a results table; RunTracker turns it into a story. Import your family's results with one click, then watch progress unfold in interactive charts you can share with anyone.

Visit live ↗

Bookmarklet import & shareable views

Parkrun has no public API, so RunTracker uses a bookmarklet: open your results page, click once, and the data flows in. Progress then becomes glassmorphism charts, and a signed link gives grandparents a read-only view — no account needed.

Key features

  • One-click bookmarklet import straight from a parkrun results page — no manual data entry.
  • Interactive charts of times, positions and progress across every event and runner.
  • Read-only sharing via signed links, so family can follow along without signing up.
  • Multi-runner support — track the whole family's parkrun journeys side by side.
  • Everything served from a single Cloudflare Worker with its database alongside.

How it's built

Front end

React + Vite + Tailwind + Recharts Rich, interactive charts and a fast SPA build.

Back end

Python Cloudflare Worker Edge compute, no server to operate.

APIs / Auth

Own REST API + share links Bookmarklet imports parkrun data; read-only views shared via signed links.

Hosting

Cloudflare Workers Global edge network, single deploy.

Database

D1 (SQLite at the edge) Colocated with the Worker — no separate DB host.
