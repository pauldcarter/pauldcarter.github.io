Hosting platforms

The hosts I've shipped on, and when I reach for each. Every platform below runs at least one of my live projects, so the pros and cons come from use, not from marketing pages.

Platform Best for Pros Cons Limits Free tier
GitHub Pages static sites & SPAs Static front ends Portfolios, docs, client-side games — anything with no server. Simple, deploy on push Lives next to the repo; HTTPS and custom domains included. Static only No server-side code, no env secrets, no database. ~1 GB site, 100 GB/mo soft Builds throttled; intended for modest traffic. Free for public repos No paid tier needed for typical use.
Cloudflare Workers edge apps & APIs Full apps at the edge APIs, SSR, real-time — with D1, KV and Durable Objects alongside. Global edge, fast cold start Storage primitives (D1, KV, Durable Objects) all in one platform. Worker runtime, not Node No full filesystem; some libraries don't run unchanged. CPU ms & bundle size caps ~3 MiB bundle cap; per-request CPU time limited. 100k req/day free D1/KV/DO have their own free allowances.
Vercel Next.js & front-end apps Next.js / SPA hosting Zero-config deploys, preview URLs per branch. Best-in-class DX Git-driven, instant previews, edge/serverless functions built in. Opinionated & usage-metered Scales with usage; heavy lock-in to its build model. Function timeout & bandwidth caps Serverless execution time and bandwidth are capped. Hobby tier free Non-commercial projects; bandwidth/build minutes capped.
Supabase Postgres backend-as-a-service Postgres + instant API Auth, auto REST/Realtime and Edge Functions over a real Postgres DB. Real Postgres, little to operate PostGIS, generated types, RLS and auth out of the box — one thing to deploy. REST shaped by PostgREST Complex queries lean on RPC; row caps mean pagination; some lock-in. PostgREST row cap Responses capped at 1000 rows — paginate with a unique sort tiebreaker. 2 projects free Plenty for small datasets; pauses after inactivity.
GCP a server you fully control Long-running stateful apps Django/Gunicorn, background jobs, anything needing a real box. Full control of a real VM Any stack, persistent disk, no platform constraints. You own the ops Patching, nginx, TLS, backups and scaling are all on you. Tiny instance 1 shared vCPU + ~1 GB RAM; scales vertically only until you pay. 1 e2-micro always-free Single region, capped egress; fine for low traffic.
